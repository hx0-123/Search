# 调试空白页面问题

如果页面显示为空白，请按以下步骤排查：

## 🔍 第一步：检查浏览器控制台

1. 打开浏览器开发者工具（F12）
2. 查看 **Console** 标签
3. 查看是否有错误信息

### 常见错误：

#### 1. Mapbox Token 未配置
```
Mapbox访问令牌未配置
```
**解决**：创建 `.env.development` 文件并配置 Token

#### 2. 组件导入错误
```
Failed to resolve import ...
```
**解决**：检查组件路径是否正确

#### 3. 路由错误
```
Cannot find module ...
```
**解决**：检查路由配置

## 🔍 第二步：检查网络请求

1. 打开 **Network** 标签
2. 刷新页面
3. 检查是否有请求失败（红色）

### 常见问题：

- **404错误**：静态资源加载失败
- **CORS错误**：后端CORS配置问题
- **500错误**：后端服务器错误

## 🔍 第三步：检查环境变量

在浏览器控制台输入：
```javascript
console.log(import.meta.env)
```

应该能看到：
- `VITE_MAPBOX_ACCESS_TOKEN`
- `VITE_API_BASE_URL`
- `VITE_WS_URL`

## 🔍 第四步：检查组件渲染

在浏览器控制台输入：
```javascript
document.querySelector('#app')
```

如果返回 `null`，说明 Vue 没有正确挂载。

## 🔍 第五步：逐步排查

### 1. 检查基本HTML结构

查看页面源代码（右键 → 查看页面源代码），应该能看到：
```html
<div id="app"></div>
```

### 2. 检查Vue是否加载

在控制台输入：
```javascript
window.Vue
```

### 3. 检查路由

访问 `http://localhost:5173/` 应该显示 HomeView
访问 `http://localhost:5173/dashboard` 应该显示 DashboardView

## 🛠️ 快速修复

### 如果完全没有内容显示：

1. **检查 main.ts 是否正确执行**
   - 在 `main.ts` 中添加 `console.log('main.ts 执行')`
   - 刷新页面，查看控制台是否有输出

2. **检查 App.vue 是否正确渲染**
   - 在 `App.vue` 的 template 中添加测试内容：
   ```vue
   <template>
     <div id="app">
       <h1>测试内容</h1>
       <router-view />
     </div>
   </template>
   ```
   - 如果能看到"测试内容"，说明 Vue 正常工作

3. **检查路由配置**
   - 临时在 `HomeView.vue` 中添加简单内容：
   ```vue
   <template>
     <div>
       <h1>HomeView 测试</h1>
     </div>
   </template>
   ```
   - 如果能看到，说明路由正常

4. **检查组件导入**
   - 逐个注释掉组件导入，找出有问题的组件

## 🚨 紧急修复

如果以上都不行，尝试：

1. **清除缓存并重新安装依赖**
   ```bash
   rm -rf node_modules
   rm package-lock.json
   npm install
   ```

2. **重启开发服务器**
   ```bash
   # 停止当前服务器 (Ctrl+C)
   npm run dev
   ```

3. **检查端口占用**
   ```bash
   # Windows
   netstat -ano | findstr :5173
   
   # Linux/Mac
   lsof -i :5173
   ```

# ⚠️ 重要提示：环境变量文件位置

## 问题

如果你的 `.env.development` 文件在 `D:\Search` 目录下，Vite **无法读取**它！

## 原因

Vite 只会在**项目根目录**（`privacy_lbs_frontend`）下查找 `.env.development` 文件，不会向上查找父目录。

## 解决方案

### ✅ 已自动复制文件

我已经尝试将文件从 `D:\Search\.env.development` 复制到 `D:\Search\privacy_lbs_frontend\.env.development`。

### 验证文件位置

请确认文件在正确位置：

```
D:\Search\privacy_lbs_frontend\.env.development  ✅ 正确位置
```

### 重启开发服务器

**重要：** 文件复制后，必须重启开发服务器才能生效！

```powershell
# 1. 停止当前服务器（按 Ctrl+C）
# 2. 重新启动
cd D:\Search\privacy_lbs_frontend
npm run dev
```

## 验证配置

重启后，打开浏览器控制台（F12），应该看到：

```
🔍 检查Mapbox配置...
Token存在: true
Token值: pk.eyJ1Ijoi...
```

如果看到这些信息，说明配置正确，地图应该能正常加载了！

## 如果仍然无法加载

1. **检查Token格式**
   - 应该以 `pk.` 开头
   - 格式：`VITE_MAPBOX_ACCESS_TOKEN=pk.eyJ1Ijoi...`

2. **检查Token有效性**
   - 登录 https://account.mapbox.com/
   - 检查Token是否有效、未过期

3. **查看浏览器控制台**
   - 按 F12 打开开发者工具
   - 查看 Console 标签页的错误信息
   - 查看 Network 标签页的请求状态

4. **清除缓存**
   ```powershell
   # 清除node_modules并重新安装
   cd D:\Search\privacy_lbs_frontend
   Remove-Item -Recurse -Force node_modules
   npm install
   npm run dev
   ```

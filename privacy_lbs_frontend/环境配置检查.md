# 环境配置检查

在启动项目前，请确保以下配置正确。

## ✅ 必需配置

### 1. Mapbox Token

**位置**：`.env.development` 文件

```env
VITE_MAPBOX_ACCESS_TOKEN=pk.your_mapbox_token_here
```

**获取方式**：
1. 访问 https://account.mapbox.com/
2. 注册/登录账号
3. 创建 Access Token
4. 复制 Token 到配置文件中

**验证**：启动项目后，地图应该能正常加载

### 2. 后端API地址

**位置**：`.env.development` 文件

```env
VITE_API_BASE_URL=http://127.0.0.1:8000/api
```

**验证**：
```bash
# 检查后端是否运行
curl http://127.0.0.1:8000/api/
```

### 3. WebSocket地址

**位置**：`.env.development` 文件

```env
VITE_WS_URL=ws://127.0.0.1:8000/ws/query_updates/
```

**验证**：启用实时更新功能时，WebSocket应该能连接

## 🔍 配置检查脚本

创建 `check_env.js` 文件（已存在）：

```javascript
// 检查环境变量配置
const fs = require('fs');
const path = require('path');

const envFile = path.join(__dirname, '.env.development');

console.log('检查环境配置...\n');

if (!fs.existsSync(envFile)) {
  console.error('❌ .env.development 文件不存在！');
  console.log('请创建 .env.development 文件并配置环境变量。');
  process.exit(1);
}

const envContent = fs.readFileSync(envFile, 'utf-8');
const envVars = {};

envContent.split('\n').forEach(line => {
  const match = line.match(/^([^=]+)=(.*)$/);
  if (match) {
    envVars[match[1].trim()] = match[2].trim();
  }
});

let hasError = false;

// 检查必需变量
const requiredVars = {
  'VITE_MAPBOX_ACCESS_TOKEN': 'Mapbox访问令牌',
  'VITE_API_BASE_URL': '后端API地址',
  'VITE_WS_URL': 'WebSocket地址',
};

for (const [key, name] of Object.entries(requiredVars)) {
  if (!envVars[key] || envVars[key].includes('your_') || envVars[key].includes('here')) {
    console.error(`❌ ${name} (${key}) 未配置或使用默认值`);
    hasError = true;
  } else {
    console.log(`✅ ${name} (${key}) 已配置`);
  }
}

if (hasError) {
  console.log('\n请检查并更新 .env.development 文件中的配置。');
  process.exit(1);
} else {
  console.log('\n✅ 所有环境变量配置正确！');
}
```

运行检查：
```bash
node check_env.js
```

## ⚠️ 常见配置问题

### 问题1：地图不显示
- **原因**：Mapbox Token未配置或无效
- **解决**：检查 `.env.development` 中的 `VITE_MAPBOX_ACCESS_TOKEN`

### 问题2：API请求失败
- **原因**：后端地址配置错误或后端未启动
- **解决**：检查 `VITE_API_BASE_URL` 和后端服务状态

### 问题3：WebSocket连接失败
- **原因**：WebSocket地址配置错误
- **解决**：检查 `VITE_WS_URL` 配置

### 问题4：环境变量不生效
- **原因**：修改环境变量后未重启开发服务器
- **解决**：停止并重新启动 `npm run dev`

## 🔄 重新加载环境变量

修改环境变量后：

1. **停止开发服务器**：`Ctrl+C`
2. **重新启动**：`npm run dev`

环境变量在构建时注入，修改后需要重启。

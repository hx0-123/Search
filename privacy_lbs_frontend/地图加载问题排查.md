# 地图加载问题排查指南

## 问题现象

地图组件显示为空白区域，没有加载出地图。

## 可能的原因和解决方案

### 1. Mapbox访问令牌未配置（最常见）

**检查方法：**
- 打开浏览器控制台（F12）
- 查看是否有错误信息："❌ Mapbox访问令牌未配置或为默认值！"

**解决方案：**

#### 步骤1：创建 `.env.development` 文件

在项目根目录 `privacy_lbs_frontend/` 下创建 `.env.development` 文件：

```ini
# Mapbox访问令牌(需去mapbox.com官网免费注册获取)
VITE_MAPBOX_ACCESS_TOKEN=pk.你的真实令牌

# 后端Django API基础地址
VITE_API_BASE_URL=http://127.0.0.1:8000/api

# WebSocket服务器地址
VITE_WS_URL=ws://127.0.0.1:8000/ws/query_updates/
```

#### 步骤2：获取Mapbox访问令牌

1. 访问 https://account.mapbox.com/
2. 注册/登录账号（免费）
3. 进入 **Access Tokens** 页面
4. 复制默认令牌或创建新令牌
5. 令牌格式类似：`pk.eyJ1IjoieW91cnVzZXJuYW1lIiwiYSI6ImN...`

#### 步骤3：更新 `.env.development` 文件

将 `pk.你的真实令牌` 替换为从Mapbox获取的真实令牌。

#### 步骤4：重启开发服务器

```powershell
# 停止当前服务器（Ctrl+C）
# 重新启动
cd D:\Search\privacy_lbs_frontend
npm run dev
```

**重要：** 修改 `.env.development` 后必须重启开发服务器才能生效！

### 2. 环境变量未正确加载

**检查方法：**
- 打开浏览器控制台
- 查看是否有 "🔍 检查Mapbox配置..." 的日志
- 查看Token是否存在和值

**解决方案：**
- 确保 `.env.development` 文件在项目根目录
- 确保变量名以 `VITE_` 开头
- 重启开发服务器

### 3. Mapbox Token无效或过期

**检查方法：**
- 控制台可能有 "地图加载错误" 的提示
- 检查Token是否在Mapbox账户中有效

**解决方案：**
- 登录Mapbox账户检查Token状态
- 创建新的Token并更新配置

### 4. 网络问题

**检查方法：**
- 检查浏览器控制台的网络请求
- 查看是否有Mapbox API请求失败

**解决方案：**
- 检查网络连接
- 检查防火墙设置
- 如果在中国大陆，可能需要使用代理

### 5. Mapbox GL CSS未加载

**检查方法：**
- 查看浏览器控制台是否有CSS加载错误

**解决方案：**
- 确保 `mapbox-gl/dist/mapbox-gl.css` 已正确导入
- 检查 `node_modules/mapbox-gl` 是否存在

## 快速诊断步骤

### 1. 检查环境变量文件

```powershell
cd D:\Search\privacy_lbs_frontend
# 检查文件是否存在
Test-Path .env.development

# 查看文件内容
Get-Content .env.development
```

### 2. 检查浏览器控制台

1. 按 F12 打开开发者工具
2. 查看 Console 标签页
3. 查找以下信息：
   - "🔍 检查Mapbox配置..."
   - "❌ Mapbox访问令牌未配置"
   - "✅ 地图加载完成！"
   - 任何错误信息

### 3. 检查网络请求

1. 在开发者工具中打开 Network 标签页
2. 刷新页面
3. 查找 mapbox 相关的请求
4. 检查请求是否成功（状态码200）


## 常见错误信息

### "Mapbox访问令牌未配置或为默认值！"
→ 需要配置真实的Mapbox Token

### "地图加载错误"
→ Token可能无效，检查Token是否正确

### "地图容器元素未找到"
→ 组件渲染问题，检查Vue组件是否正确挂载

## 成功标志

配置正确后，应该看到：
- ✅ 浏览器控制台显示 "✅ 地图加载完成！"
- ✅ 页面显示完整的地图界面
- ✅ 可以缩放、拖拽地图
- ✅ 显示导航控件和全屏按钮

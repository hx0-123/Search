# 立即修复步骤

## 问题确认

控制台显示：
- `Token存在: false` ❌
- `Token值: 未设置` ❌

这说明环境变量没有被Vite读取。

## 立即执行以下步骤

### 步骤1：重新创建环境变量文件

在项目目录 `D:\Search\privacy_lbs_frontend` 下，执行：

```powershell
cd D:\Search\privacy_lbs_frontend

# 运行修复脚本
.\fix_env.ps1
```

或者手动创建：

```powershell
# 删除旧文件
Remove-Item .env.development -ErrorAction SilentlyContinue

# 创建新文件（替换为你的真实Token）
$token = "pk.你的真实Mapbox令牌"  # 替换这里！

@"
VITE_MAPBOX_ACCESS_TOKEN=$token
VITE_API_BASE_URL=http://127.0.0.1:8000/api
VITE_WS_URL=ws://127.0.0.1:8000/ws/query_updates/
"@ | Out-File -FilePath .env.development -Encoding utf8
```

**重要：**
- 确保Token是真实的（从 https://account.mapbox.com/ 获取）
- Token格式：`pk.eyJ1Ijoi...`（以pk.开头）
- 不要有引号
- 等号前后不要有空格

### 步骤2：清除Vite缓存

```powershell
cd D:\Search\privacy_lbs_frontend
Remove-Item -Recurse -Force node_modules\.vite -ErrorAction SilentlyContinue
```

### 步骤3：重启开发服务器

```powershell

```

### 步骤4：清除浏览器缓存并刷新

1. 按 **Ctrl + Shift + Delete** 清除浏览器缓存

## 如果仍然显示 Token存在: false

### 检查1：文件位置

确认文件在正确位置：
```
D:\Search\privacy_lbs_frontend\.env.development  ✅
```

不是：
```
D:\Search\.env.development  ❌
```

### 检查2：文件内容格式

打开 `.env.development` 文件，确认格式：

```ini
# ✅ 正确格式
VITE_MAPBOX_ACCESS_TOKEN=pk.eyJ1IjoieW91cnVzZXJuYW1lIiwiYSI6ImN...
VITE_API_BASE_URL=http://127.0.0.1:8000/api
VITE_WS_URL=ws://127.0.0.1:8000/ws/query_updates/
```

**错误格式：**
```ini
# ❌ 有引号
VITE_MAPBOX_ACCESS_TOKEN="pk.xxx"

# ❌ 有空格
VITE_MAPBOX_ACCESS_TOKEN = pk.xxx

# ❌ 变量名错误（缺少VITE_前缀）
MAPBOX_ACCESS_TOKEN=pk.xxx
```

### 检查3：文件编码

在VS Code中：
1. 打开 `.env.development` 文件
2. 查看右下角编码显示
3. 如果不是UTF-8，点击编码 → "通过编码保存" → 选择"UTF-8"

### 检查4：在浏览器控制台测试

重启服务器后，在浏览器控制台运行：

```javascript
// 检查所有环境变量
console.log('所有环境变量:', import.meta.env);

// 检查Token
console.log('Token:', import.meta.env.VITE_MAPBOX_ACCESS_TOKEN);
console.log('Token存在:', !!import.meta.env.VITE_MAPBOX_ACCESS_TOKEN);

// 列出所有VITE_开头的变量
Object.keys(import.meta.env).filter(k => k.startsWith('VITE_')).forEach(k => {
    console.log(`${k}:`, import.meta.env[k]);
});
```

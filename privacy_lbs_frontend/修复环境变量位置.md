# 修复环境变量文件位置

## 问题说明

**重要：** Vite 只能在**项目根目录**（`privacy_lbs_frontend`）下读取 `.env.development` 文件，不能从父目录读取！

如果你的 `.env.development` 文件在 `D:\Search` 目录下，需要将其移动到或复制到 `D:\Search\privacy_lbs_frontend` 目录下。

## 快速修复

### 方法1：复制文件到项目目录（推荐）

```powershell
# 如果文件在父目录，复制到项目目录
Copy-Item "D:\Search\.env.development" "D:\Search\privacy_lbs_frontend\.env.development"
```

### 方法2：手动移动文件

1. 在文件管理器中找到 `D:\Search\.env.development`
2. 剪切文件
3. 粘贴到 `D:\Search\privacy_lbs_frontend\` 目录

### 方法3：在项目目录重新创建

如果父目录的文件内容正确，可以在项目目录重新创建：

```powershell
cd D:\Search\privacy_lbs_frontend

# 查看父目录的文件内容
Get-Content "D:\Search\.env.development"

# 然后手动创建或使用以下命令（替换为你的真实Token）
@"
# Mapbox访问令牌
VITE_MAPBOX_ACCESS_TOKEN=pk.你的真实令牌

# 后端Django API基础地址
VITE_API_BASE_URL=http://127.0.0.1:8000/api

# WebSocket服务器地址
VITE_WS_URL=ws://127.0.0.1:8000/ws/query_updates/
"@ | Out-File -FilePath .env.development -Encoding utf8
```

## 验证文件位置

运行检查脚本：

```powershell
cd D:\Search\privacy_lbs_frontend
.\check_env.ps1
```

或者手动检查：

```powershell
# 检查项目目录
Test-Path "D:\Search\privacy_lbs_frontend\.env.development"

# 查看文件内容
Get-Content "D:\Search\privacy_lbs_frontend\.env.development"
```

## 文件位置要求

```
D:\Search\
├── .env.development          ❌ 错误位置（Vite无法读取）
├── privacy_lbs_frontend\
│   ├── .env.development      ✅ 正确位置（Vite可以读取）
│   ├── src\
│   ├── package.json
│   └── ...
└── ...
```

## 重启开发服务器

**重要：** 移动或创建文件后，必须重启开发服务器：

```powershell
# 1. 停止当前服务器（Ctrl+C）
# 2. 重新启动
cd D:\Search\privacy_lbs_frontend
npm run dev
```

## 验证配置

重启后，打开浏览器控制台（F12），应该看到：

```
🔍 检查Mapbox配置...
Token存在: true
Token值: pk.eyJ1Ijoi...
✅ 地图加载完成！
```

如果仍然有问题，检查：
1. Token格式是否正确（应该以 `pk.` 开头）
2. Token是否有效（登录Mapbox账户检查）
3. 浏览器控制台是否有其他错误信息

# 地图未加载问题排查步骤

## 当前状态

文件 `.env.development` 已在项目目录下，但地图仍未加载。

## 排查步骤

### 步骤1：检查文件内容

打开 `.env.development` 文件，确认内容格式正确：

```ini
# 正确的格式
VITE_MAPBOX_ACCESS_TOKEN=pk.eyJ1IjoieW91cnVzZXJuYW1lIiwiYSI6ImN...

# 错误的格式（这些都不会工作）
VITE_MAPBOX_ACCESS_TOKEN=pk.your_mapbox_token_here
VITE_MAPBOX_ACCESS_TOKEN=pk.你的真实令牌
VITE_MAPBOX_ACCESS_TOKEN=your_token_here
```

**重要检查点：**
- ✅ Token必须以 `pk.` 开头
- ✅ Token必须是真实的Mapbox令牌（从 https://account.mapbox.com/ 获取）
- ✅ 不能有引号：`VITE_MAPBOX_ACCESS_TOKEN="pk.xxx"` ❌
- ✅ 不能有空格：`VITE_MAPBOX_ACCESS_TOKEN = pk.xxx` ❌

### 步骤2：检查浏览器控制台

1. 按 **F12** 打开开发者工具
2. 切换到 **Console** 标签页
3. 查看是否有以下信息：

```
🔍 检查Mapbox配置...
Token存在: true/false
Token值: pk.xxx... 或 未设置
```

**如果看到：**
- `Token存在: false` → 环境变量未加载
- `Token值: pk.your_mapbox_token_here` → Token还是默认值
- `Token值: 未设置` → 环境变量未读取

### 步骤3：确认已重启开发服务器

**重要：** 修改 `.env.development` 文件后，**必须重启开发服务器**才能生效！

```powershell
# 1. 停止当前服务器（按 Ctrl+C）
# 2. 重新启动
cd D:\Search\privacy_lbs_frontend
npm run dev
```

### 步骤4：在浏览器控制台测试

打开浏览器控制台（F12），运行以下代码：

```javascript
// 检查环境变量
console.log('Token:', import.meta.env.VITE_MAPBOX_ACCESS_TOKEN);
console.log('Token存在:', !!import.meta.env.VITE_MAPBOX_ACCESS_TOKEN);
console.log('所有环境变量:', import.meta.env);
```

### 步骤5：检查文件编码

确保 `.env.development` 文件使用 **UTF-8** 编码：

1. 在VS Code中打开文件
2. 查看右下角的编码显示
3. 如果不是UTF-8，点击编码 → 选择"通过编码保存" → 选择"UTF-8"

### 步骤6：验证Token有效性

1. 访问 https://account.mapbox.com/
2. 登录账户
3. 进入 **Access Tokens** 页面
4. 确认Token：
   - ✅ 状态为"Active"
   - ✅ 未过期
   - ✅ 有正确的权限

## 常见问题

### Q1: Token存在但地图还是不显示？

**可能原因：**
- Token无效或已过期
- Token权限不足
- 网络问题（防火墙、代理）

**解决方案：**
1. 在Mapbox账户中创建新Token
2. 更新 `.env.development` 文件
3. 重启开发服务器

### Q2: 重启后还是读取不到环境变量？

**可能原因：**
- 文件位置不对
- 文件格式有问题
- Vite缓存问题

**解决方案：**
```powershell
# 清除Vite缓存
cd D:\Search\privacy_lbs_frontend
Remove-Item -Recurse -Force node_modules\.vite -ErrorAction SilentlyContinue
npm run dev
```

### Q3: 控制台显示Token存在，但值不对？

**检查：**
- 文件内容是否有引号
- 是否有多余的空格
- 是否有多行（应该在一行）

**正确格式示例：**
```ini
VITE_MAPBOX_ACCESS_TOKEN=pk.eyJ1IjoieW91cnVzZXJuYW1lIiwiYSI6ImNsb2Z0a2F1d2UwMDBjMXNqY2V1a2EifQ.xxxxxxxxxxxxx
```

## 快速诊断命令

在项目目录运行：

```powershell
cd D:\Search\privacy_lbs_frontend

# 检查文件是否存在
Test-Path .env.development

# 查看文件内容（隐藏敏感信息）
Get-Content .env.development | ForEach-Object {
    if ($_ -match "VITE_MAPBOX_ACCESS_TOKEN") {
        $token = $_ -replace "VITE_MAPBOX_ACCESS_TOKEN=", ""
        Write-Host "Token前20字符: $($token.Substring(0, [Math]::Min(20, $token.Length)))..."
        Write-Host "Token格式正确: $($token -match '^pk\.')"
    } else {
        Write-Host $_
    }
}
```

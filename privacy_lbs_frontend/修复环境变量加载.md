# 修复环境变量未加载问题

## 问题诊断

控制台显示：
- `Token存在: false`
- `Token值: 未设置`

这说明 `.env.development` 文件存在，但Vite没有读取到环境变量。

## 可能的原因

1. **文件格式问题**（最常见）
   - 文件编码不是UTF-8
   - 文件有BOM标记
   - 换行符问题

2. **文件内容格式错误**
   - 有引号
   - 有空格
   - 变量名错误

3. **Vite缓存问题**
   - 需要清除缓存

4. **未重启开发服务器**
   - 修改.env文件后必须重启

## 解决方案

### 方案1：重新创建文件（推荐）

删除旧文件，重新创建：

```powershell
cd D:\Search\privacy_lbs_frontend

# 删除旧文件
Remove-Item .env.development -ErrorAction SilentlyContinue

# 创建新文件（替换为你的真实Token）
@"
VITE_MAPBOX_ACCESS_TOKEN=pk.你的真实令牌
VITE_API_BASE_URL=http://127.0.0.1:8000/api
VITE_WS_URL=ws://127.0.0.1:8000/ws/query_updates/
"@ | Out-File -FilePath .env.development -Encoding utf8 -NoNewline
```

**重要：**
- 使用 `-Encoding utf8` 确保UTF-8编码
- 使用 `-NoNewline` 避免末尾换行问题
- Token值不要有引号
- 等号前后不要有空格

### 方案2：检查并修复现有文件

```powershell
cd D:\Search\privacy_lbs_frontend

# 查看文件内容（检查格式）
Get-Content .env.development

# 检查文件编码
$content = [System.IO.File]::ReadAllBytes('.env.development')
$encoding = [System.Text.Encoding]::Default
Write-Host "文件编码可能需要检查"
```

### 方案3：清除Vite缓存并重启

```powershell
cd D:\Search\privacy_lbs_frontend

# 清除Vite缓存
Remove-Item -Recurse -Force node_modules\.vite -ErrorAction SilentlyContinue

# 重启开发服务器
npm run dev
```

## 验证步骤

### 1. 确认文件格式

文件内容应该是这样（**无引号，无空格**）：

```ini
VITE_MAPBOX_ACCESS_TOKEN=pk.eyJ1IjoieW91cnVzZXJuYW1lIiwiYSI6ImN...
VITE_API_BASE_URL=http://127.0.0.1:8000/api
VITE_WS_URL=ws://127.0.0.1:8000/ws/query_updates/
```

**错误示例：**
```ini
# ❌ 有引号
VITE_MAPBOX_ACCESS_TOKEN="pk.xxx"

# ❌ 有空格
VITE_MAPBOX_ACCESS_TOKEN = pk.xxx

# ❌ 变量名错误
MAPBOX_ACCESS_TOKEN=pk.xxx  # 缺少VITE_前缀
```

### 2. 在浏览器控制台测试

重启服务器后，在浏览器控制台运行：

```javascript
// 检查环境变量
console.log('所有环境变量:', import.meta.env);
console.log('Token:', import.meta.env.VITE_MAPBOX_ACCESS_TOKEN);
console.log('Token存在:', !!import.meta.env.VITE_MAPBOX_ACCESS_TOKEN);
```

### 3. 检查Vite配置

确保 `vite.config.ts` 没有特殊配置阻止环境变量加载。

## 快速修复脚本

运行以下命令自动修复：

```powershell
cd D:\Search\privacy_lbs_frontend

# 备份旧文件
if (Test-Path .env.development) {
    Copy-Item .env.development .env.development.backup
}

# 读取Token（如果有）
$token = ""
if (Test-Path .env.development) {
    $content = Get-Content .env.development
    $tokenLine = $content | Where-Object { $_ -match "VITE_MAPBOX_ACCESS_TOKEN" }
    if ($tokenLine -match "=(.+)") {
        $token = $matches[1].Trim() -replace '["\''\s]', ''
    }
}

# 创建新文件
if ($token -and $token -notmatch "your_token|你的真实") {
    @"
VITE_MAPBOX_ACCESS_TOKEN=$token
VITE_API_BASE_URL=http://127.0.0.1:8000/api
VITE_WS_URL=ws://127.0.0.1:8000/ws/query_updates/
"@ | Out-File -FilePath .env.development -Encoding utf8 -NoNewline
    Write-Host "✅ 文件已重新创建（保留Token）" -ForegroundColor Green
} else {
    @"
VITE_MAPBOX_ACCESS_TOKEN=pk.你的真实令牌
VITE_API_BASE_URL=http://127.0.0.1:8000/api
VITE_WS_URL=ws://127.0.0.1:8000/ws/query_updates/
"@ | Out-File -FilePath .env.development -Encoding utf8 -NoNewline
    Write-Host "⚠️  文件已创建，请填入真实的Mapbox Token" -ForegroundColor Yellow
}

Write-Host ""
Write-Host "下一步："
Write-Host "1. 如果Token是默认值，请替换为真实Token"
Write-Host "2. 重启开发服务器（Ctrl+C 然后 npm run dev）"
Write-Host "3. 清除浏览器缓存并刷新页面"
```

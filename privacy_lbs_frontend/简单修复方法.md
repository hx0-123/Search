# 简单修复方法

## 问题

控制台显示 `Token存在: false`，说明环境变量没有被读取。

## 最简单的解决方法

### 方法1：直接在VS Code中编辑文件

1. **打开文件**
   - 在VS Code中打开 `D:\Search\privacy_lbs_frontend\.env.development`

2. **检查并修改内容**
   确保文件内容格式如下（**无引号，无空格**）：
   ```
   VITE_MAPBOX_ACCESS_TOKEN=pk.你的真实令牌
   VITE_API_BASE_URL=http://127.0.0.1:8000/api
   VITE_WS_URL=ws://127.0.0.1:8000/ws/query_updates/
   ```

3. **保存文件**
   - 按 `Ctrl + S` 保存
   - 确保右下角显示编码为 **UTF-8**

4. **重启开发服务器**
   ```powershell
   # 停止服务器（Ctrl+C）
   # 重新启动
   cd D:\Search\privacy_lbs_frontend
   npm run dev
   ```

### 方法2：使用PowerShell命令（一行）

```powershell
cd D:\Search\privacy_lbs_frontend; "VITE_MAPBOX_ACCESS_TOKEN=pk.你的真实令牌`nVITE_API_BASE_URL=http://127.0.0.1:8000/api`nVITE_WS_URL=ws://127.0.0.1:8000/ws/query_updates/" | Out-File -FilePath .env.development -Encoding utf8 -NoNewline
```

**注意：** 将 `pk.你的真实令牌` 替换为真实的Mapbox Token。

### 方法3：检查现有文件内容

如果文件已存在，检查内容：

```powershell
cd D:\Search\privacy_lbs_frontend
Get-Content .env.development
```

**常见问题：**
- 如果看到引号：`VITE_MAPBOX_ACCESS_TOKEN="pk.xxx"` → 删除引号
- 如果看到空格：`VITE_MAPBOX_ACCESS_TOKEN = pk.xxx` → 删除空格
- 如果Token是默认值：`pk.your_mapbox_token_here` → 替换为真实Token

## 关键检查点

### ✅ 正确格式
```
VITE_MAPBOX_ACCESS_TOKEN=pk.eyJ1IjoieW91cnVzZXJuYW1lIiwiYSI6ImN...
```

### ❌ 错误格式
```
VITE_MAPBOX_ACCESS_TOKEN="pk.xxx"          # 有引号
VITE_MAPBOX_ACCESS_TOKEN = pk.xxx          # 有空格
VITE_MAPBOX_ACCESS_TOKEN=pk.your_mapbox_token_here  # 默认值
MAPBOX_ACCESS_TOKEN=pk.xxx                 # 缺少VITE_前缀
```

## 验证步骤

1. **检查文件位置**
   ```
   D:\Search\privacy_lbs_frontend\.env.development  ✅
   ```

2. **检查文件内容**
   - 打开文件查看格式是否正确

3. **清除缓存并重启**
   ```powershell
   cd D:\Search\privacy_lbs_frontend
   Remove-Item -Recurse -Force node_modules\.vite -ErrorAction SilentlyContinue
   npm run dev
   ```

4. **检查浏览器控制台**
   - 按 F12
   - 查看 Console
   - 应该看到 `Token存在: true`

## 如果仍然不行

在浏览器控制台运行：

```javascript
// 检查所有环境变量
console.log('All env vars:', import.meta.env);

// 检查VITE_开头的变量
Object.keys(import.meta.env)
  .filter(k => k.startsWith('VITE_'))
  .forEach(k => console.log(k, ':', import.meta.env[k]));
```

如果仍然没有 `VITE_MAPBOX_ACCESS_TOKEN`，可能是：
1. 文件编码问题（需要UTF-8）
2. 文件位置不对
3. Vite配置问题

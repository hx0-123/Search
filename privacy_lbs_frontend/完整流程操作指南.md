# 完整流程操作指南

本指南将帮助您完成从启动到测试的完整流程。

## 📋 前置准备

### 1. 检查环境

确保已安装：
- Node.js (>= 20.19.0 或 >= 22.12.0)
- npm 或 yarn
- Python 3.x (用于后端)
- Redis (用于Celery，如需要)

### 2. 配置环境变量

在 `privacy_lbs_frontend` 目录下创建 `.env.development` 文件：

```env
# Mapbox访问令牌
VITE_MAPBOX_ACCESS_TOKEN=pk.your_mapbox_token_here

# 后端API基础地址
VITE_API_BASE_URL=http://127.0.0.1:8000/api

# WebSocket服务器地址
VITE_WS_URL=ws://127.0.0.1:8000/ws/query_updates/
```

**重要**：请将 `your_mapbox_token_here` 替换为您的实际 Mapbox Token。

## 🚀 启动步骤

### 第一步：启动后端服务

```bash
# 进入后端目录
cd ../privacy_lbs_backend

# 激活虚拟环境（Windows）
venv\Scripts\activate

# 激活虚拟环境（Linux/Mac）
source venv/bin/activate

# 启动Django开发服务器
python manage.py runserver

# 如果使用Celery，在另一个终端启动Worker
python -m celery -A privacy_lbs_backend worker --loglevel=info --pool=solo
```

后端服务将在 `http://127.0.0.1:8000` 启动。

### 第二步：启动前端服务

```bash
# 进入前端目录
cd privacy_lbs_frontend

# 安装依赖（如果还没安装）
npm install

# 启动开发服务器
npm run dev
```

前端服务将在 `http://localhost:5173` 启动。

### 第三步：验证服务运行

1. **检查后端**：访问 `http://127.0.0.1:8000/api/` 应该能看到API响应
2. **检查前端**：访问 `http://localhost:5173` 应该能看到应用界面

## 🧪 完整功能测试流程

### 测试场景1：基础查询流程

1. **打开应用**
   - 访问 `http://localhost:5173`
   - 应该看到地图和侧边栏

2. **获取当前位置**
   - 点击"获取当前位置"按钮
   - 浏览器会请求位置权限，点击"允许"
   - 地图应该移动到您的位置
   - 查询点标记应该显示在地图上

3. **输入查询文本**
   - 在"查询文本"输入框中输入关键词（如"餐厅"、"咖啡"等）
   - 观察表单验证状态

4. **调整查询参数**
   - 调整权重滑块（alpha值）
   - 设置结果数量（K值）
   - 可选：设置查询半径

5. **发起查询**
   - 点击"发起查询"按钮
   - 观察加载状态
   - 等待查询结果返回

6. **查看结果**
   - 结果应该显示在侧边栏的结果列表中
   - 地图上应该显示结果标记（绿色圆点）
   - 点击列表项，地图应该高亮对应标记
   - 点击地图标记，列表应该选中对应项

### 测试场景2：权重调整和前端排序

1. **在已有结果的情况下**
   - 调整权重滑块
   - 如果勾选了"权重调整时自动重新查询"，应该触发新查询
   - 如果未勾选，应该进行前端重新排序

2. **验证排序**
   - 在结果列表中选择不同的排序方式（评分/距离/名称）
   - 验证排序是否正确

### 测试场景3：实时位置更新

1. **启用实时更新**
   - 在查询成功后，找到"实时位置更新"卡片
   - 打开实时更新开关

2. **模拟位置变化**
   - 在地图上拖拽查询点标记
   - 或者点击"获取当前位置"更新位置
   - 观察是否触发位置更新

3. **检查更新效果**
   - 查看更新次数和最后更新时间
   - 观察轨迹线是否绘制
   - 检查结果是否更新

### 测试场景4：地图交互

1. **地图缩放**
   - 使用鼠标滚轮或地图控件缩放
   - 如果结果很多，应该看到聚类效果

2. **地图拖拽**
   - 拖拽地图移动视图
   - 查询点和结果标记应该跟随移动

3. **标记交互**
   - 点击结果标记，查看弹出窗口
   - 点击聚类点，地图应该放大

### 测试场景5：路线规划

1. **选择多个结果**
   - 在结果列表中点击多个结果项
   - 点击"路线规划"按钮

2. **查看路线详情**
   - 应该打开路线详情抽屉
   - 显示路线概览、途经点、详细步骤

3. **地图显示路线**
   - 点击"在地图上显示"按钮
   - 路线应该显示在地图上（如果已实现）

## 🔍 调试技巧

### 使用浏览器开发者工具

1. **Network标签**
   - 查看所有API请求
   - 检查请求参数和响应数据
   - 查看请求状态码

2. **Console标签**
   - 查看日志信息
   - 查看错误信息
   - 开发环境下会显示详细的调试信息

3. **Vue Devtools**（如果已安装）
   - 检查组件状态
   - 查看Pinia store数据
   - 调试Vue组件

### 常见问题排查

#### 问题1：地图不显示
- **检查**：Mapbox Token是否正确配置
- **检查**：浏览器控制台是否有错误
- **检查**：网络连接是否正常

#### 问题2：API请求失败
- **检查**：后端服务是否运行
- **检查**：CORS配置是否正确
- **检查**：API地址是否正确
- **检查**：Network标签查看具体错误

#### 问题3：查询无结果
- **检查**：后端是否有数据
- **检查**：查询参数是否正确
- **检查**：后端日志查看处理过程

#### 问题4：实时更新不工作
- **检查**：WebSocket连接是否建立
- **检查**：更新间隔设置
- **检查**：是否有查询ID

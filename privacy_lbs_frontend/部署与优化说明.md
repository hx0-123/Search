## 🔧 与后端联调

### 1. 启动后端服务

确保Django后端服务已启动，并配置了CORS：

```python
# settings.py
CORS_ALLOWED_ORIGINS = [
    "http://localhost:5173",  # Vite开发服务器
    "http://127.0.0.1:5173",
]
```

### 2. 启动前端开发服务器

```bash
cd privacy_lbs_frontend
npm run dev
```

前端将在 `http://localhost:5173` 启动。

### 3. 检查API请求/响应

使用浏览器开发者工具（Network Console）和Vue Devtools：

- **Network标签**: 检查API请求的URL、参数、响应数据
- **Console标签**: 查看错误信息和日志
- **Vue Devtools**: 检查组件状态和Pinia store

### 4. 测试连续查询流程

1. 发起查询 → 检查返回结果格式
2. 移动位置（拖拽查询点或更新位置） → 检查是否触发新查询或返回缓存结果
3. 调整权重 → 检查是否重新查询或前端重新排序
4. 启用实时更新 → 检查位置更新频率和防抖效果

## 🚀 部署

### 1. 构建生产版本

```bash
cd privacy_lbs_frontend
npm run build
```

构建产物在 `dist/` 目录下。

### 2. 环境变量配置

创建 `.env.production` 文件：

```env
VITE_MAPBOX_ACCESS_TOKEN=your_production_mapbox_token
VITE_API_BASE_URL=https://your-backend-domain.com/api
VITE_WS_URL=wss://your-backend-domain.com/ws/query_updates/
```

### 3. 部署到Nginx

#### Nginx配置示例

```nginx
server {
    listen 80;
    server_name your-domain.com;
    
    root /path/to/privacy_lbs_frontend/dist;
    index index.html;
    
    # 前端路由支持
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    # API代理（可选）
    location /api {
        proxy_pass http://your-backend:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
    
    # WebSocket代理（可选）
    location /ws {
        proxy_pass http://your-backend:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
    
    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

### 4. Docker部署（可选）

创建 `Dockerfile`:

```dockerfile
FROM node:20-alpine AS builder

WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

构建和运行：

```bash
docker build -t privacy-lbs-frontend .
docker run -d -p 80:80 privacy-lbs-frontend
```

## ⚠️ 关键注意事项

### 1. 安全红线和API设计

**最重要**：前端绝不接触任何原始密文或密钥！

- ✅ API返回的 `SafeZone`、`TopKResults` 等都应该是后端完成所有安全计算和权限验证后，生成的、仅供展示用的明文数据
- ✅ 前端只负责展示和交互，不进行任何加密解密操作
- ✅ 所有敏感操作（加密、解密、密钥管理）都在后端完成

### 2. 错误处理

- ✅ 已实现统一的错误处理工具：`src/utils/error-handler.util.ts`
- ✅ API错误、地图错误、位置获取错误都有友好的用户提示
- ✅ 开发环境下会在控制台打印详细错误信息

### 3. 移动端适配

- ✅ 响应式布局：侧边栏在移动端自动变为顶部
- ✅ 触摸交互：地图支持触摸操作
- ⚠️ 建议：进一步优化移动端UI，考虑使用移动端专用的地图控件

### 4. 浏览器兼容性

- 支持现代浏览器（Chrome、Firefox、Safari、Edge）
- 需要支持ES6+和WebGL（用于Mapbox）

### 5. 性能监控

建议添加性能监控：

```typescript
// 在 main.ts 中添加
if (import.meta.env.PROD) {
  // 发送性能指标到监控服务
  window.addEventListener('load', () => {
    const perfData = performance.getEntriesByType('navigation')[0];
    console.log('页面加载时间:', perfData.loadEventEnd - perfData.fetchStart);
  });
}
```

## 🔍 常见问题

### Q: 地图不显示？
A: 检查Mapbox Token是否正确配置，查看浏览器控制台错误信息。

### Q: API请求失败？
A: 检查CORS配置、API地址、网络连接。

### Q: 实时更新不工作？
A: 检查WebSocket连接、更新间隔设置、防抖配置。

### Q: 构建后路由404？
A: 确保Nginx配置了 `try_files` 指令支持前端路由。

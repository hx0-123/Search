# PowerShell 激活虚拟环境说明

## 问题

在PowerShell中直接运行 `venv\Scripts\activate` 可能会遇到错误：
```
无法加载模块"venv"。有关详细信息，请运行"Import-Module venv"。
```

## 解决方案

### 方法1：使用完整路径（推荐）

```powershell
.\venv\Scripts\Activate.ps1
```

或者：

```powershell
venv\Scripts\Activate.ps1
```

### 方法2：使用批处理文件

```powershell
venv\Scripts\activate.bat
```

### 方法3：修改PowerShell执行策略（一次性设置）

如果方法1和方法2都不行，需要修改执行策略：

```powershell
# 以管理员身份运行PowerShell，然后执行：
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

然后就可以使用：
```powershell
.\venv\Scripts\Activate.ps1
```

### 方法4：使用CMD（最简单）

如果不想修改执行策略，可以直接使用CMD：

```cmd
venv\Scripts\activate.bat
```

## 快速启动命令（PowerShell）

### 启动Django后端

```powershell
cd D:\Search\privacy_lbs_backend
.\venv\Scripts\Activate.ps1
python manage.py runserver
```

### 启动Celery Worker

```powershell
cd D:\Search\privacy_lbs_backend
.\venv\Scripts\Activate.ps1
python -m celery -A privacy_lbs_backend worker --loglevel=info --pool=solo
```

## 验证虚拟环境已激活

激活成功后，命令提示符前面会显示 `(venv)`：

```
(venv) PS D:\Search\privacy_lbs_backend>
```

## 退出虚拟环境

```powershell
deactivate
```
